---
title: "Revoke API Key"
api: "DELETE https://api.bland.ai/keys/delete"
description: "Revoke an API key. The revocation is immediate."
---

## Overview

Revokes the specified API key. Once revoked, the key can no longer authenticate any request.

The caller must have permission to revoke the key (the key's owning user or an admin of the owning org).

Note: this endpoint lives at `/keys/delete`, not `/v1/keys/delete`.

---

## Headers

<ParamField header="authorization" type="string" required>
  Your API key for authentication. Cannot be the same key you are trying to revoke.
</ParamField>

---

## Body Parameters

<ParamField body="key" type="string" required>
  The 14-character key identifier to revoke. This is the **first 14 characters** of the API key string (for example `org_abcdef1234`), not the full key. Passing the full key returns `Invalid key`.

  Key identifiers can be fetched from [List API Keys](/api-v1/post/keys-list), which returns the full `key` value; trim it to 14 characters before passing here.
</ParamField>

---

## Response

Returns `200 OK` with an empty payload on success.

<ResponseField name="data" type="null">
  Always `null` on success.
</ResponseField>

<ResponseField name="errors" type="null | array">
  `null` on success.
</ResponseField>

<ResponseExample>
```json Response
{
  "data": null,
  "errors": null
}
```

```json Missing Key
{
  "data": null,
  "errors": [
    { "error": "API_KEY_DELETE_FAILURE", "message": "Missing key in request body" }
  ]
}
```

```json Invalid Key
{
  "data": null,
  "errors": [
    { "error": "API_KEY_DELETE_FAILURE", "message": "Failed to delete key: Invalid key" }
  ]
}
```
</ResponseExample>
