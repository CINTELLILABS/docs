---
title: "List API Keys"
api: "POST https://api.bland.ai/keys/list"
description: "List the API keys scoped to your user or to an org you have permission for."
---

## Overview

Returns the API keys that exist under a given scope. Two scopes are supported:

- `user`, lists keys owned by the calling user (across any org).
- `org`, lists keys owned by a specific org. Requires `target_id` to identify the org.

The full key values are returned in the response, treat them like passwords. To revoke a returned key, trim it to its first 14 characters and pass that to [Revoke API Key](/api-v1/delete/keys-delete).

Note: this endpoint lives at `/keys/list`, not `/v1/keys/list`. The router is mounted without the `/v1/` prefix.

---

## Headers

<ParamField header="authorization" type="string" required>
  Your API key for authentication. The caller's identity is resolved from this key and is used to gate access to the requested scope.
</ParamField>

---

## Body Parameters

<ParamField body="type" type="string" required>
  Scope to list. One of `user` or `org`.
</ParamField>

<ParamField body="target_id" type="string">
  Required when `type` is `org`. UUID of the org whose keys you want to list. Must be an org you have permission to read keys for.
</ParamField>

---

## Response

<ResponseField name="data.keys" type="array">
  Array of API keys.

  <Expandable title="Key fields">
    <ResponseField name="key" type="string">
      The full API key value.
    </ResponseField>
    <ResponseField name="owner" type="string">
      UUID of the user or org that owns the key.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="errors" type="null | array">
  `null` on success.
</ResponseField>
