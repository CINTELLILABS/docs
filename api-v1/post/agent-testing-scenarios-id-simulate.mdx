---
title: "Simulate Scenario"
api: "POST https://api.bland.ai/v1/agent-testing/scenarios/{id}/simulate"
description: "Start an agent-to-agent simulation run for a scenario. Returns 202 immediately with the simulation run identifier while conversations are generated and scored in the background."
---

Use this endpoint to evaluate an agent against a scenario without placing real calls. Bland generates the requested number of simulated conversations between a tester persona and your agent, then scores every conversation with each eval agent attached to the scenario.

Attach the eval agents you want to use for scoring first with [Attach Eval Agent to Scenario](/api-v1/post/agent-testing-scenarios-id-eval-agents).

<Note>
  This endpoint returns `202 Accepted` immediately after validating the request and creating the simulation run. Generation and scoring continue in the background. Poll [Get Simulation Run Results](/api-v1/get/agent-testing-simulation-runs-id-results) with the returned `simulation_set_id` to track progress and read final results.
</Note>

### Headers

<ParamField header="authorization" type="string" required>
  Your API key for authentication.
</ParamField>

### Path Parameters

<ParamField path="id" type="string" required>
  The scenario ID to simulate.
</ParamField>

### Body Parameters

<ParamField body="simulations_count" type="integer">
  Number of conversations to generate and score. Must be a positive integer. Defaults to `5` and is capped at `50`.
</ParamField>

<ParamField body="version_number" type="integer">
  Pathway version to simulate against. Defaults to the current production version.
</ParamField>

### Response

Returns `202 Accepted` as soon as the simulation run has been created.

<ResponseField name="simulation_set_id" type="string">
  Identifier for the simulation run. Use this value as the `id` for [Get Simulation Run Results](/api-v1/get/agent-testing-simulation-runs-id-results) and [List Simulation Runs for Scenario](/api-v1/get/agent-testing-scenarios-id-simulation-runs).
</ResponseField>

<ResponseField name="status" type="string">
  Initial status of the simulation run. Always `RUNNING` on this response.
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST https://api.bland.ai/v1/agent-testing/scenarios/a1b2c3d4-5678-9abc-def0-1234567890ab/simulate \
  -H "authorization: $BLAND_API_KEY" \
  -H "content-type: application/json" \
  -d '{
    "simulations_count": 10
  }'
```
</RequestExample>

<ResponseExample>
```json Response
{
  "simulation_set_id": "d4e5f6a7-89ab-cdef-0123-4567890abcde",
  "status": "RUNNING"
}
```
</ResponseExample>
