---
title: "List Simulation Runs for Scenario"
api: "GET https://api.bland.ai/v1/agent-testing/scenarios/{id}/simulation-runs"
description: "List recent simulation runs for a scenario, newest first, each with the same summary shape returned by Get Simulation Run Results."
---

Use this endpoint to poll the recent simulation runs for a scenario — for example, to power a runs panel in your own dashboard alongside [Simulate Scenario](/api-v1/post/agent-testing-scenarios-id-simulate).

Each entry uses the same summary shape as [Get Simulation Run Results](/api-v1/get/agent-testing-simulation-runs-id-results). While a run is still generating or scoring, `eval_run_id` is `null` and `status` is `RUNNING`.

### Headers

<ParamField header="authorization" type="string" required>
  Your API key for authentication.
</ParamField>

### Path Parameters

<ParamField path="id" type="string" required>
  The scenario ID.
</ParamField>

### Response

Returns an array of simulation run summaries ordered newest-first, capped at the 20 most recent runs. Returns `[]` when the scenario has never been simulated.

<ResponseField name="simulation_set_id" type="string">
  The simulation run ID. Pass this as the `id` for [Get Simulation Run Results](/api-v1/get/agent-testing-simulation-runs-id-results).
</ResponseField>

<ResponseField name="eval_run_id" type="string">
  The linked eval run ID. `null` when generation finished but no eval run exists yet.
</ResponseField>

<ResponseField name="status" type="string">
  Eval run status (`PENDING`, `RUNNING`, `COMPLETE`, `PARTIAL`, `FAILED`, or `CANCELLED`) — or `RUNNING` while the linked eval run does not exist yet.
</ResponseField>

<ResponseField name="score" type="number">
  Aggregate run score from 0 to 100, rounded to one decimal place. `null` while pending or when nothing has been scored.
</ResponseField>

<ResponseField name="passed" type="boolean">
  Whether the run passed overall. `null` while in progress or unscored.
</ResponseField>

<ResponseField name="total_calls" type="integer">
  Total conversations generated for the run.
</ResponseField>

<ResponseField name="passed_count" type="integer">
  Number of conversations whose call status finished as `COMPLETE`.
</ResponseField>

<ResponseField name="failed_count" type="integer">
  Number of conversations that failed or produced insufficient evidence.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the run was created.
</ResponseField>
