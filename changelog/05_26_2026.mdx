### Call Transfers in Personas

Personas now have a dedicated Call Transfers panel under the Behavior tab, so you can set up human handoffs without leaving the persona builder.

- **Trigger prompt** — describe in plain language when the agent should hand off (for example, "when the caller asks for a human" or "after three failed authentication attempts"). The more specific the prompt, the fewer false positives.
- **Default transfer number** — the fallback destination in E.164 format used when no transfer list label matches.
- **Transfer list** — add labeled destinations (like `billing` or `sales`) the agent can pick from based on the conversation. Each entry pairs a label with a phone number.
- **Cold or warm transfer** — cold transfer hands the caller off immediately. Warm transfer dials the live agent first, briefs them, then merges the caller in. Warm transfer is Enterprise-only.

Open a persona, go to the Behavior tab, and select **Configure Transfers** to set everything up. The panel summarizes the current configuration at a glance and stays in sync with persona versioning, so transfer changes follow the same draft → promote flow as the rest of the persona.
