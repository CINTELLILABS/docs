### Early hangup message on warm transfer

Warm Transfer nodes now support an optional **Early Hangup Message** that plays to the live agent when the caller disconnects after the agent has answered but before the calls have merged. Instead of an abrupt silent drop, the agent hears a short explanation and then the call ends cleanly.

- Set the message on the Transfer Call node's warm transfer configuration to give your live agents context when a caller hangs up mid-briefing
- Only triggers in the pre-merge window after the agent has answered — caller disconnects before pickup or after a successful merge are unaffected
- Leave the field blank to preserve the existing immediate-teardown behavior

See the full configuration reference in the [Warm Transfer tutorial](/tutorials/warm-transfer#11-early-hangup-message).
