### Route testing in the testbed

The pathway testbed now supports **Route** as a fourth test type, alongside dialogue, loop condition, and variable extraction. Use it to confirm that a router node still picks the correct outgoing branch after you edit route descriptions, add a new route, or change the conversation flow leading into it.

- Open expanded call logs on any pathway call and hover a route entry to reveal the testbed button
- The testbed opens pre-loaded with the full conversation up to the routing decision, the router's current configuration, and the originally chosen route name as the expected output
- Run simulations to see how often the router picks the same route under varying conditions, then publish the result as a [Standard](/tutorials/standards) to lock in the behavior

### Redesigned testbed editor

The testbed editor has been rebuilt around a sectioned view that surfaces exactly the fields each test type needs, so you can focus on the part of the prompt you're iterating on without scrolling through the full message array.

- Each test type shows the relevant sections — node prompt for dialogue tests, loop condition for condition tests, variables for extraction tests, available routes for route tests — plus the global prompt and conversation history
- Toggle **Raw** in the editor header to drop into the underlying messages when you need to edit something the sectioned view doesn't expose
- Click **Reset** to instantly revert every edit back to the original prompt and conversation history

See [Testbed](/tutorials/testbed) for the full workflow.
