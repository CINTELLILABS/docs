## Introduction

Personas are unified AI agents that manage all your phone numbers and use cases in one place. Instead of configuring each phone number separately with different prompts and settings, create a single intelligent persona that handles everything - from smart call routing to maintaining consistent personality across all interactions.

<CardGroup cols={2}>
  <Card title="Smart Routing" icon="route">
    Personas automatically route calls to the right pathways based on conversation context, like routing billing questions to your billing pathway.
  </Card>
  <Card title="Visual Builder" icon="palette">

## Building Your Persona

The persona builder is organized into five main sections, each handling different aspects of your AI agent:

### General Configuration

Define your persona's identity, voice settings, and communication channels.

<img style={{ borderRadius: "0.5rem" }} src="/tutorials/tutorials-assets/personas/create_persona.jpeg" />

**Key Settings:**
- **Identity**: Set your persona's name, role, and description
- **Voice Selection**: Choose from available voices (like "June", "Karl", or "Estella")
- **Language**: Select primary language
- **Background Noise**: Add ambient sounds like "Office-style soundscape" with typing, chatter, and office sounds
- **Modalities**: Configure communication channels:
  - Voice & Calls (with "Add to Inbound Numbers" integration)

  - Web Widget (coming soon)

#### Voicemail handling

The **Voicemail** section on the Behavior tab controls what the persona does when a call reaches voicemail. This is the source of voicemail behavior for every call the persona handles, including outbound calls dispatched by automations through `persona_id`.

Select a **Voicemail Action**:

- **Hangup** — end the call as soon as voicemail is detected. This is the default.
- **Ignore** — continue the call as if no voicemail was detected. Useful for IVRs or numbers where the initial audio can look like a voicemail greeting.
- **Leave Message** — play a voicemail message after the beep, then end the call. Enter the text to speak in the **Voicemail Message** field that appears.
- **Leave SMS** — send an SMS instead of leaving a voice message. Enter the text to send in the **Voicemail SMS** field that appears.

The message and SMS fields only appear for the action they belong to. Switching to a different action clears the field for the previous one, so the saved configuration always matches the selected action.

<Note>
Automations that call a persona (for example, a Trigger's **Send Call** action referencing a `persona_id`) resolve the persona's **production** version. After changing the voicemail action, use **Promote to Production** so live automation calls pick up the new setting.
</Note>

### Analysis & Webhooks

Set up post-conversation analysis and webhook integrations.

<img style={{ borderRadius: "0.5rem" }} src="/tutorials/tutorials-assets/personas/persona_analysis.jpeg" />

**Analysis Configuration:**
- **Generate Summary**: Toggle to automatically create short summaries after each conversation with customizable summary prompt
- **Data Extraction**: Configure citation schemas for extracting specific information from calls (Enterprise only)
- **Outcomes**: Select one or more [Outcomes](/tutorials/outcomes) to run against every call this persona handles. Selected outcomes are attached to each call automatically, so you don't need to pass `disposition_ids` per request, and the results are available in post-call automations and webhooks.
- **Evaluations**: Attach a published [Eval](/tutorials/evals) workbench setup to this persona to score every call it handles. See [Attaching an eval to a persona](#attaching-an-eval-to-a-persona) below.
- **Webhook Integration**:
  - URL field for webhook endpoint (e.g., "https://example.com/webhook")
  - Events dropdown to select which webhook events will be sent to your endpoint during the call
  - "Test Webhook" button for validation

#### Attaching an eval to a persona

Use the **Evaluations** selector in the Analysis tab to pick a published [workbench setup](/tutorials/evals#workbench-setups). Every call this persona handles is then scored automatically once the call ends, and verdicts show up alongside your other [Eval](/tutorials/evals) results.

**Behavior:**
- Only **published** workbench setups appear in the dropdown. The attachment pins to the published version, so editing the setup later does not affect already-completed calls.
- Selecting an eval **auto-enables call recording**, because evals require a recorded call. If you later turn recording off while an eval is still attached, an inline warning shows and the eval will not run.
- If the attached setup is later unpublished, the persona still shows it with an "unpublished" warning. Republish the setup or pick a different one to resume scoring.
- The selection is saved to the persona **draft**. Live calls keep using the production version until you **Promote to Production**.

To remove an eval, clear the **Evaluations** selector and save the persona.

## Testing Your Persona

Before you deploy your persona, use the built-in testing interface to make sure it's working as expected. On the right side of the persona builder, you'll find a testing modal where you can start web calls with yourself using the current status of your persona. This lets you test changes before they go live.

## API Usage

Use your personas programmatically through the Bland API for automated workflows.

### Basic API Integration

Reference your persona in API calls using the `persona_id` parameter:

```json
{
  "phone_number": "+15551234567",
  "persona_id": "your_persona_id_here",
  "task": "Additional context or overrides"
}
```

### Finding Your Persona ID

Use the List Personas API endpoint to retrieve your persona IDs:

```bash
curl -X GET "https://api.bland.ai/v1/personas" \
  -H "Authorization: YOUR_API_KEY_HERE"
```

This returns all your personas with their IDs, names, and configuration details. Copy the `id` field from the persona you want to use.

You can also try this directly from our [API docs playground](https://docs.bland.ai/api-v1/get/personas?playground=open).

### Parameter Override Behavior

When using `persona_id` in API calls:
- Persona settings provide the base configuration
- Any additional parameters in your API request override the persona defaults
- This allows for flexible, context-specific customization while maintaining consistency
