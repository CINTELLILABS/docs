## Introduction

**Conversational Pathway** is Bland AI’s visual flow builder for designing, controlling, testing, and launching AI-powered phone calls.

If you want more control over how your AI handles conversations, Pathway lets you build structured dialogue flows using connected steps instead of relying only on a single prompt. Each step represents a stage in the conversation, and you can decide how the AI moves from one step to another based on user responses. This gives you full control over how conversations branch, where they go next, and how they end.

Pathway supports different types of steps, including actions like ending a call, transferring a call to a human representative, sending SMS, triggering a webhook during the call, accessing a knowledge base, running custom code, and more. You can capture key information during the call, connect to external APIs in real time, route conversations based on live data, and even set global steps that can activate from anywhere in the flow.

There are also built-in testing tools that let you test specific parts of the conversation, simulate different paths, and review detailed logs before going live so you can confidently deploy your AI.

# Node Types

Within Bland’s conversational pathways, you can utilize an array of different nodes. Below are the node types referenced in the source.

## Default Node

<img
  style={{ borderRadius:"0.5rem" }}
  src="/tutorials/tutorials-assets/default_node_icon.png"
/>

A **Default Node** generates a response to the user.

#### Dialogue Prompt

Located under the Default Node. Instructions for what the agent should say.

> **Note:** Enable **Static Text** to set a fixed response the agent always says.

## Large Text Node

<img
  style={{ borderRadius:"0.5rem" }}
  src="/tutorials/tutorials-assets/large_text_node.png"
/>

A variant of the Default node designed for handling longer text content in the conversation.

## Transfer Call Node

The Transfer Node is used to route a live call from the AI to a human agent or an external phone number.

It allows you to seamlessly hand off the conversation when human intervention is required.

### Examples of When to Use the Transfer Node

- The user requests to speak with a human.
- The issue requires manual handling.
- A high-value lead needs a sales representative.
- The AI is unable to resolve the request.

There are two types of transfer calls:

### 1. Cold Transfer (Standard Transfer)

The user is immediately transferred to another phone number without additional context being shared beforehand.

### 2. Warm Transfer (Enterprise Feature)

The AI initiates a separate call to brief the human agent on the customer’s situation before connecting the user to the agent.

Learn more about the [Warm Transfer Enterprise Feature](https://docs.bland.ai/tutorials/warm-transfer)

## Webhook Node

<img
  style={{ borderRadius:"0.5rem" }}
  src="/tutorials/tutorials-assets/laweb_hook_node.png"
/>

Webhook nodes allow you to make calls to external services during your call. Many customers use these nodes to fetch things like customer data from Salesforce, or deal information from Hubspot, or even time availability from Google Calendar.

**Note:** If you want data to be sent after the call ends we can set up a Post Call Webhook that Bland AI automatically sends to your server after a phone call completes.

Learn more about [Post Call Webhook](https://docs.bland.ai/tutorials/post-call-webhooks)

## Transfer Pathway Node

Hands the conversation off to a different pathway (supports dynamic routing via variables).

## Press Button Node

Simulates pressing a phone keypad button (DTMF tone), with optional “Go to Node” action.

## IVR Node

Navigates automated phone menus (IVR systems) by pressing buttons and listening for prompts.

## Knowledge Base Node

Responds to user questions by referencing the text or data provided in the knowledge base attached to the node.

## End Call Node

Ends the call after saying a final message (e.g., goodbye).

## Wait for Response Node

Pauses and waits for the user to respond before continuing.

## Custom Tool Node (Legacy)

Executes a pre-configured custom tool (external API integration) from your Tools Library within the pathway.

Check out [Custom Tool](https://docs.bland.ai/tutorials/tools) to know more about it!

## Route Node

Directs the conversation to different nodes based on defined conditions, without generating a spoken response.

Note that Advanced Pathway Settings will be deprecated based from the [May 19, 2025 changelog](https://docs.bland.ai/changelog/05_19_2025#deprecation-of-advanced-pathway-settings).

## SMS Node (Enterprise)

Sends an outbound SMS message during the call (Enterprise feature).

Check out [SMS](https://docs.bland.ai/tutorials/sms) to know more about it!

## Twilio Flow Redirect Node

Redirects the call to a Twilio Studio Flow for additional processing.

## Scheduling Node

Fetches available time slots from your system via webhook, helps the caller pick a time, and confirms the appointment.

## Amazon Connect Node (Enterprise)

Transfers the call into an Amazon Connect contact flow.

## Transfer SMS Chat Node (Beta / Enterprise)

Contact us for Beta access! Transfers an SMS conversation to another pathway or handler. (Enterprise feature)

## Custom Code Node (Enterprise)

Runs custom JavaScript code for advanced logic, calculations, or API calls (Enterprise feature).

# Node Library

Allows you to reuse commonly used nodes in multiple pathways, or use optimized nodes shared by the community for certain use cases.

Open your pathway and click **Add New Node** and select the tab **Node Library** to see the Public Library or add your own Node (Only for your own Organization).

3. **Insert:** Add any block directly onto the canvas with "Add to Pathway"

# Variables

Reference variables in prompts using double curly braces: `{{variable_name}}`.

## Built-in Variables

Built-in Variables are automatically available, no setup needed. These variables are extracted _during_ the call from the conversation itself. In pathway nodes (like webhook or scheduling nodes), you define variable names, types, and descriptions, and the AI extracts values from what was discussed.

Here are some examples of Built-in Variables:

- `{{phone_nnumber}}` — Other party’s number
- `{{timezone}}` — Timezone of the other party
- `{{country}}` — Country code
- `{{state}}` — State abbreviation
- `{{city}}` — Full city name
- `{{zip}}` — Zip code
- `{{call_id}}` — Unique call ID
- `{{now}}` — Current time in call’s timezone
- `{{now_utc}}` — Current time in UTC
- `{{from}}` — Outbound number in E.164 format
- `{{to}}` — Inbound number in E.164 format
- `{{short_from}} / {{short_to}}` — Numbers without country code
- `{{lastUserMessage}}` — Last user response
- `{{prevNodePrompt}}` — Previous node’s prompt

## Call into Variables (request_data)

Call into variables are variables you pass into a call at the time of sending it via the `request_data` field in the API request. These are not built-in. They would be custom variables you pass via request_data or extract during conversation.

They're accessible in prompts and pathways using `{{variable_name}}` syntax.

Some examples are:

- `{{party_size}}`
- `{{reservation_date}}`
- `{{reservation_time}}`

# Other Node Terminology

Within most Bland nodes, there are 3 main LLM functions that control the flow of the conversation:

- Dialogue Generation
- Loop Conditions
- Variable Extractions

## Dialogue Prompt

The Dialogue Prompt sets the main goal for what your agent will say at that point in the conversation. Think of it as the script or instruction that guides your agent's response during a specific step of the call.

## Loop Condition

- Controls when the agent can exit the current node.
- The agent remains on the node until the condition is satisfied.
- Includes a likelihood score (e.g., 7/10) evaluated each turn.

**Example:**\
“Wait until the user provides a valid time and the agent confirms it.”

## Variable Extraction Prompts

Pulls structured information from the conversation into named variables.

These variables can be used in:

- Webhook request bodies
- Prompts
- Subsequent nodes

Syntax for use: `{{variable_name}}`

## Global Node

- A node configured with: `isGlobal: true`
- And a `globalLabel` (e.g., “user asks a question”).
- Can be triggered from any point in the pathway.
- Used for handling interruptions such as:
  - FAQs
  - Off-topic questions
  - Requests to speak to a human

## Unit Tests

Validate dialogue prompts, variable extraction, and loop logic on individual nodes using historical call data. You can generate user input permutations to stress-test prompts, and pin calls to auto-run with every test. This is part of the Standards framework for regression testing.

## Note Tag / Tag Name / Tag Color

Tags are metadata labels assigned to nodes or calls.

- Tag Name → The label text
- Tag Color → Visual identifier

Used for:

- Organizing calls
- Filtering call logs
- Categorizing pathways

Example filter:

- “Includes tag”
- “Does not include tag”

## Temperature

Controls the randomness/creativity of the model's responses at this node.

- Lower = More deterministic
- Higher = More creative

## Dynamic Interruption Threshold

Controls the sensitivity to user interruptions at this specific node. Adjusts how easily the user can interrupt the agent mid-speech.

## Skip User’s Response

When toggled on, the agent proceeds to the next node without waiting for the user to respond.

## Block Interruptions

Prevents the user from interrupting the agent while it's speaking on this node. The agent will finish its full response before listening.

## Disable Repeating on Silence

When enabled, the agent will not repeat itself if the user is silent. Normally the agent may re-prompt after silence.

## Condition overrides global pathway

When enabled, this node's edge/routing conditions take priority over any global node conditions. Prevents global nodes from intercepting when this node's specific conditions should apply.

## Disable Recording

Stops call recording at this node for PCI compliance. Recording automatically resumes when the conversation exits the node.

## Disable Logging

Stops transcript logging at this node for sensitive data handling. Like recording, it resumes automatically after leaving the node.

# Standards

Standards let you build better agents, faster, by defining scenarios, backtesting prompts against them, and grading how those prompts perform.

See more about standards here: https://docs.bland.ai/tutorials/standards#overview-of-standards

## Pathway (Edges)

Pathways are the connectors between nodes and determine which next node the agent can move to, usually determined by how the user responds to your assistant.

## Routing

Refers to how the AI agent navigates between nodes in a Conversational Pathway.

### LLM Based Routing

The LLM evaluates the user's response against the available edge labels and picks the best match.

For example, from a "Start" node, you might have edges labeled:

- "User wants to reserve"
- "Live Agent Requested"

The agent chooses based on the context of the user response.

The agent makes decisions based on labels you put in the pathways when connecting one node to another. (You can also include pathway descriptions for best results.)

### Logic Based Routing

Handles branching logic and conditional navigation within pathways, determined by a variable which could be a request_data or call-into variable.

From our example pathway: We can use our call-into variable **party_size** to determine if the agent will tell the user that they have a 20% discount since they are more than 4 pax.

Simply click the Pathway and click Advanced Settings to enable Logic Based Routing!

> Important: This method of variable comparison is being deprecated. We now recommend using Route Nodes for more powerful and flexible conditions.

Benefits of Route Nodes:

- Enables global node support (unlike current variable comparison)
- Allows fallback paths
- Provides a central place to manage all routes and conditions

The dialogue generated will be based on the instructions you set in the nodes.

You can manually create a pathway from a node by clicking the grey circle at the bottom of the node and dragging your mouse to connect to the top grey circle of another node.

# Pathway Test

## Chat

The most common way to test. Type messages in the input field and the agent responds as it would on a live call. Each message displays a timestamp. As the conversation progresses, you can track which node the agent moves to next.

Steps:

1. Click the **Test** button on upper right corner
2. Click **Start text Chat**

You can even branch the conversation mid-test to explore different scenarios without starting over.

Read more about Conversation Branching: [February 09, 2026 Changelog](https://docs.bland.ai/changelog/02_09_2026#conversation-branching)

Example:

1. User wants to talk or says yes → Continue Chat Node is chosen (Default)
2. User does not want to talk → Quick Goodbye Node is chosen (End Call)

## Pathway Test using Call

Triggers a real phone call using your pathway configuration. The call behaves identically to a production call, and logs appear in the testing panel in real time.

Steps:

1. Click the **Test** button on upper right corner
2. Navigate to **CALL** tab
3. Add From Phone Number (optional)
4. Add which Phone Number to call (required)

Example:

User wants book a reservation → Collect Reservation Details Node is chosen →\
User mentions there are more than 4 people → 20% discount for 5 pax node is chosen.

## Pathway Logs

Show the step-by-step progression of a call through your pathway nodes.

The agent starts at the first node, and then moves to the next node based on the pathway that the agent decides to take. The agent will then execute the instructions in the node as dialogue, and then move on to the next node based on the pathway that the agent decides to take. This process will continue until the agent decides to end the call.

## Pathway Test using Web Chat

An embeddable chat interface that lets you add AI-powered conversations directly to your website. It can be powered by either a pathway (using `pathway_id`) or a simple agent prompt (using `agent_prompt`).

# Global Prompt

Applies to all nodes in this pathway. Use them for consistent behavior, context, or guidelines. Include details such as the Agent’s name, main goal, how they interact, etc. and keep this prompt short to reduce the likelihood of hallucinations.

Example:

“You are a professional and friendly AI receptionist for a dental office named Sarah. Your goal is to help patients schedule, reschedule, or inquire about dental appointments efficiently while maintaining a warm and reassuring tone.

Speak clearly, calmly, and confidently. Keep responses concise and easy to understand.

Always:

- Be polite, empathetic, and professional.
- Confirm key details such as patient name, date of birth (if required), appointment type, preferred date, and preferred time.
- Repeat important information back to the patient before finalizing an appointment.
- Ask follow-up questions if required information is missing.
- Guide the conversation step-by-step instead of asking multiple complex questions at once.
- If the patient expresses anxiety about treatment, respond reassuringly and calmly.
- If you do not know something, politely say you will check or that a staff member can assist.”

# Pathway Settings (Top-right Dropdown)

## Enable/Disable Memory

Toggles memory on a pathway so the agent remembers previous conversations across all calls per user. Found in the top-right dropdown menu next to "Send Call" in the pathway editor.

Source: [**/tutorials/memories**](https://docs.bland.ai/tutorials/memories)

## Enable Advanced Mode

Toggles detailed execution logs showing secondary actions like loop conditions, variable extractions, and routing during pathway testing.

Source: [February 9, 2026 changelog](https://docs.bland.ai/changelog/02_09_2026)

## Export Pathway

Downloads your pathway as a JSON file containing all nodes, edges, and configurations. This allows you to back up, version-control, or transfer pathways between organizations. You can re-import exported JSON files when creating a new pathway.

Source: [February 16, 2026 changelog](https://docs.bland.ai/changelog/02_16_2026)

## Share Pathway

Generates a shareable link or submits your pathway to the Pathway Showcase, allowing other users or team members to view or use your pathway as a template.

Source: [September 8, 2025 changelog](https://docs.bland.ai/changelog/09_08_2025)

## Test Cases (Legacy)

A testing interface within the pathway editor where you can simulate conversations against your pathway. Features include:

- Choose a start node, version, and request data
- Branch conversations from any message to explore alternate scenarios
- Save configurations for reuse
- Run tests against historic calls using drafted node changes
- Use candidate models before promoting

Source: [February 9, 2026 changelog](https://docs.bland.ai/changelog/02_09_2026)

## Call Logs (Legacy)

Displays a filterable, searchable log of all calls made using this pathway. Includes call details like transcripts, recordings, duration, extracted variables, pathway version used, and a MEMORY tab showing contact facts/entities captured. You can also export call logs and filter by pathway tags, phone numbers, and timestamps.

Source: [January 20, 2026 changelog](https://docs.bland.ai/changelog/01_20_2026)
